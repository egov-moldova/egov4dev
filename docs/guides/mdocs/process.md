## Prerequisites:  

The beneficiary requesting integration must meet the following conditions: 

1. It has an information system for integration with MDocs; 
2. The information system is registered in the [Register of Information Resources and Systems](https://rsi.gov.md/procedure); 
3. The contract and/or annex regarding the use of the MDocs service has been signed; 
4. If the institution does not have a system certificate, it must be requested and obtained from [STISC](https://semnatura.md/order/system-certificate). 
5. The institution must send the system certificate (.cer public key) to AGE at servicii@egov.md only if it does not already hold a valid certificate registered for another AGE platform service. 
6. If the institution already holds an active certificate, it must send the certificate’s serial number to AGE to confirm that it is the same certificate used for another service. 
 
## Connection steps: 

1. Fill in the [Unified Connection Form](https://forms.office.com/pages/responsepage.aspx?id=Z4f8jWsRaEKDxfvIWTRtOHwLku9yOJZAs22J7gTUxBNUMFhISEJPMUNGOTIwV09OTkFTUUtLSk9LTS4u&route=shorturl); 
2. Sign the contract and the MDocs service provision annex;<br>Note! If the institution already has a system integrated with another AGE service and has an active contract, it is not necessary to sign a new contract, but only to sign the MDocs service provision annex.
3. Send the following information to AGE at servicii@egov.md ; 
    - the .cer file (public key), if the institution does not have a system certificate;
    - or the certificate’s serial number, if the institution already has an valid certificate;
    - the name of the requesting system;
    - the public IP address;<br>**Note!** MDocs only allows access from registered IP addresses or via a secure VPN. 
5. AGE configures the institution's access to the staging environment; 
6. AGE creates and sets the document types required for integration; 
7. The beneficiary integrates their system with the staging environment;  
8. After successful testing, the beneficiary submits the confirmation to AGE; 
9. AGE deploys the service in the production environment; 
10. The beneficiary is notified about the successful integration.
