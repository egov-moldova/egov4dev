## Prerequisites:  

A beneficiary, requesting integration with MNotify, shall meet the following conditions conditions: 

1. To own an information system to be integrated with MNotify; 
2. For public beneficiaries – the system shall be registered with the [Registrul Resurselor și Sistemelor Informaționale](https://rsi.gov.md/procedure) (Recommended);  
3. To have concluded a Contract and/or Annex to use MNotify service; 
4. For private beneficiaries – this service shall be paid according to the contract concluded; 
5. Beneficiaries must hold a valid system certificate issued by STISC for integration;<br>f the institution does not yet hold a system certificate, it must request and obtain one from STISC. If the institution already holds an active certificate used for integration with another AGE platform service (e.g., MPass, MSign, MDocs), the same certificate may be reused, and no new certificate is required.
6. To configure the system certificate in MPass.

## Connection steps for the beneficiary: 

1. To fill in the Formularul unic de conectare;  
2. To sign the contract and/or annex for providing the governmental electronic notification service (MNotify)<br>**Note:** If the institution already has an active contract for another AGE platform service, it is not necessary to sign a new contract. In this case, only the specific service annex for integration (e.g., MPass, MSign, MDocs) must be signed.
3. The institution must send the system certificate (.cer public key) to AGE at [servicii@egov.md](servicii@egov.md) only if it does not already hold a valid certificate registered for another AGE platform service. If the institution already holds an active certificate, it must send the certificate’s serial number to AGE to confirm that it is the same certificate used for another service.;  
4. To fill in the integration data in the [Form](https://forms.office.com/Pages/DesignPageV2.aspx?origin=NeoPortalPage&subpage=design&id=Z4f8jWsRaEKDxfvIWTRtODb3-31Q6N1Hrz_Ou4oWQqBUNlpMVU02NThQU1oxQko5T0I1WkVMN0dQSi4u) 
    - Name of information system in MPass; 
    - IP addresses: 
      - for testing environment - Ex. 172.46.45.10 
      - for production environment - Ex. 172.46.45.11 
    - Details regarding the notification channels to be used 
5. EGA configures access on the testing environment; 
6. The beneficiary performs the operational tests in compliance with the Guide; 
7. Upon the beneficiary’s final validation, EGA is to activate the service on the production environment; 
8. The beneficiary shall be notified when the integration completed.
